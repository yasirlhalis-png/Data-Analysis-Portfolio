# Urban Dubai Restaurant Data Analysis

## Project Overview
This project focuses on data wrangling, cleaning, and analysis of a messy restaurant dataset consisting of 100+ records. The goal was to transform raw, inconsistent data into a structured format ready for business intelligence.

## Data Cleaning Process
The raw data contained various inconsistencies, which were handled through:
* **Date Standardization:** Converted multiple date formats to a uniform (DD-MM-YYYY) format.
* **Text Normalization:** Fixed typos in Cuisine (e.g., 'Chineese' to 'Chinese') and standardized dish names.
* **Data Transformation:** Converted word-based quantities (e.g., 'two') to numeric values (2) and corrected 'NaN' or non-numeric prices.
* **Categorization:** Filled missing category labels and standardized Payment Modes.
* **Deduplication:** Identified and removed exact and near-duplicates to ensure data integrity.

## Key Insights Discovered
* **Top Revenue Generators:** Identified high-performing dishes like [Biryani/Manchurian].
* **Payment Trends:** Analyzed customer preference between Cash, Card, and Online payments.
* **Cuisine Popularity:** Calculated average spending per cuisine to understand customer behavior.

* Data Validation & Formulaic Correction
